🎵 사운드 매니저 (SoundManager) 사용 가이드

안녕하세요 여러분! 게임의 타격감과 분위기를 책임질 SoundManager 세팅이 완료되었습니다.
방치형 게임 특성상 수십 개의 효과음이 터져도 렉이 걸리지 않도록 '라운드 로빈' 방식의 오브젝트 풀링을 적용해 최적화를 마쳤습니다.

별도의 세팅 없이 씬 어디서나 아래 코드 한 줄만 복사해서 쓰시면 됩니다!

1️⃣ 효과음 (SFX) 재생하기

몬스터 피격, 공격, 버튼 클릭 등 짧게 끝나는 효과음을 낼 때 사용합니다.

// 기본 사용법
SoundManager.Instance.PlaySFX(재생할_오디오_클립);


💡 (꿀팁) 찰진 타격감을 위한 피치(Pitch) 랜덤화 기능!

무기를 휘두르거나 몬스터가 맞을 때, 똑같은 소리가 계속 나면 귀가 피로해집니다.
이때 뒤에 true를 하나 붙여주면, 매니저가 알아서 음높이를 미세하게 바꿔주어 타격감이 2배 상승합니다!

// 전투 타격음에 적극 권장!
SoundManager.Instance.PlaySFX(타격음_클립, true);


2️⃣ 배경음악 (BGM) 재생 및 정지

스테이지 돌입, 보스전 진입 시 사용합니다. BGM은 기본적으로 무한 반복(Loop) 됩니다.

// 브금 틀기
SoundManager.Instance.PlayBGM(보스전_브금_클립);

// 브금 끄기 (사망 연출 등)
SoundManager.Instance.StopBGM();


3️⃣ 볼륨(Volume) 조절 (환경설정 UI용)

유저가 슬라이더를 통해 볼륨을 조절할 때 사용합니다. Define.Sound Enum을 활용합니다.
(볼륨 값은 0.0f ~ 1.0f 사이입니다)

// 배경음악 볼륨 50%로 줄이기
SoundManager.Instance.SetVolume(Define.Sound.Bgm, 0.5f);

// 효과음 볼륨 끄기(음소거)
SoundManager.Instance.SetVolume(Define.Sound.Effect, 0.0f);


❓ 개발 지식: '라운드 로빈(Round Robin) 풀링'이 뭔가요?

방치형 RPG는 플레이어의 광역 스킬 하나에 몬스터 10마리가 동시에 맞으면서 1초에도 수십 번씩 타격음이 겹쳐서 발생하는 특징이 있습니다.

❌ 기존 방식의 문제점: 소리가 날 때마다 오디오 오브젝트를 생성(Instantiate)하고 파괴(Destroy)하면, 메모리 쓰레기(가비지 컬렉션, GC)가 미친 듯이 쌓여서 게임이 뚝뚝 끊기는 렉이 발생합니다. 일반적인 오브젝트 풀(Object Pool)을 쓰더라도 매번 켜고 끄는(SetActive) 연산 비용이 아깝습니다.

✅ 라운드 로빈 풀링 (우리가 적용한 방식):

게임이 켜질 때 딱 **15개의 오디오 소스(방)**를 미리 만들어 둡니다.

소리를 내라고 할 때마다 1번 방, 2번 방, 3번 방... 순서대로 돌아가면서 소리를 틉니다.

15번 방까지 다 썼다면? 다시 1번 방으로 돌아가서 이전 소리 위에 새로운 소리를 덮어씌워 틉니다!

👨‍💻 결론: 수거할 필요도, 껐다 켤 필요도 없는 초고속 최적화 기법입니다. Instantiate나 Destroy를 전혀 사용하지 않으므로 메모리 누수나 프레임 드랍이 절대 발생하지 않습니다. 광역 스킬 타격음 등에 마음껏 남발(?)하셔도 좋습니다!
